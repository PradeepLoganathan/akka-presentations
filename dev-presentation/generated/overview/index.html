The premise

Governance is the first and last topic in AI.

AI systems make probabilistic judgments. Trust comes from defining how those judgments are scored, constrained, tested, and proven.

The shift

Deterministic systems made governance a bolt-on. AI makes it structural.

Deterministic software
  • Behavior mostly specified in code
  • Tests assert expected outcomes
  • Governance arrives through review, audit, policy
  • Compliance often surrounds the system
AI systems
  • Behavior emerges from model, prompt, context, tools
  • Quality must be scored across dimensions
  • Controls need to exist at runtime and in CI/CD
  • Governance defines the system posture
Enterprise consequence: governance teams gain real authority over AI delivery.
The trust model

Trust requires a closed loop: score, control, regress.

ScoreEvery probabilistic decision across the dimensions that matter.
->
ControlBlock, escalate, gate, alert, or observe based on those scores.
->
RegressUse scored history to prove the system is improving or not degrading.

Controls are not one thing

EvalMatrix rows compile to different control loci.

Runtime

Guardrails, Evaluations, tool and LLM boundary decisions.

Human flow

HITL workflow pauses and HOTL human-sourced observations.

Build / CI

TestKit, EvalKit, regression suites, blocking gates.

Deployment

Sign-off, posture checks, materiality gates, exceptions.

Operations

Timers, event-triggered Evaluations, drift signals, alerts.

Evidence

Interaction logs, audit records, coverage reports, retained proof.

Human judgment

HITL and HOTL are different governance placements.

HITL

Application flow pause

  • Workflow waits for a human decision
  • Used when the business process requires human ownership
  • Resumes application logic with a feedback payload
HOTL

Human as Evaluation source

  • Human produces an observation
  • Used by Evaluations, guardrails, monitoring, calibration
  • Feeds scores, labels, thresholds, and evidence
Placement matters because it determines who owns the decision and where proof is emitted.
