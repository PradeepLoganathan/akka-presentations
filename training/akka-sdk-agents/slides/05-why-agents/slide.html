Why agents belong here

An agent is a stateful, addressable, reactive LLM caller.

That's a mouthful. Look at each word — you already have infrastructure for it.

Stateful

Session memory persists across turns. Same runtime the entities use.

Addressable

Session id routes you to the same agent instance. Same sharding as entities.

Reactive

Agents can be called from workflows, consumers, endpoints — anywhere a component can.

You don't bolt an LLM onto Akka. You host it as a component and get the whole runtime for free.
