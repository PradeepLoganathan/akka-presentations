Memory: three built-in strategies.

ProviderBehaviour
DisabledNo history. Each turn is a cold start.
LimitedWindowBounded history - last N messages or K bytes. Sample uses this.
FromConfigReads memory settings from application.conf. Central overrides.
CustomPlug in your own SessionMemory implementation. Redis, external store, whatever.
Memory is persisted by the runtime - the same cluster mechanism as entities. No external cache needed.
