Agents

Session id is the router key.

Same session id → same memory. Different sessions → isolated conversations.

SupportEndpoint
@Post("/support/{sessionId}")
public Answer ask(String sessionId, Question req) {
  String answer = client.forAgent()
      .inSession(sessionId)
      .method(SupportAgent::ask)
      .invoke(req.question());
  return new Answer(sessionId, answer);
}
  • Caller chooses the session id — per-user, per-ticket, per-thread
  • forAgent().inSession(id) is how ComponentClient routes
  • Memory is scoped to the session — not the agent
  • Two different agents in the same session share memory
  • Sessions never explicitly end — memory bounds them naturally
