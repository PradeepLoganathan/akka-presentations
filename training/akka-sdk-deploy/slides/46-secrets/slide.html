Deploy

Secrets: env vars, not code.

Terminal
# create a secret
akka secrets create generic openai-key \
    --literal OPENAI_API_KEY=sk-...

# wire it into the service's env
akka services deploy akka-workshop \
    ghcr.io/pradeepl/akka-workshop:1.0 \
    --secret-env openai-key

# rotate
akka secrets update generic openai-key \
    --literal OPENAI_API_KEY=sk-new...
akka services rolling-update akka-workshop
  • Secrets never appear in the image, config, or logs
  • Mounted as env vars - picked up by application.conf substitution
  • Rotate via update + rolling-update
  • Access control: only the service can see its wired secrets
  • Audit trail on every read (see runbook slide 62)
