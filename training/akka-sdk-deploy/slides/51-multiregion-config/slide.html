Multi-region

Config-first: no code changes.

Terminal
# create a project spanning three regions
akka projects new akka-workshop-prod \
    --region gcp-us-east1 \
    --region gcp-europe-west1 \
    --region gcp-asia-southeast1

# set primary region for a service
akka services deploy akka-workshop \
    ghcr.io/pradeepl/akka-workshop:1.0 \
    --region gcp-us-east1

# add a replica region
akka services replicate akka-workshop \
    --to gcp-europe-west1
  • Regions are project-level; services opt in
  • Sample code compiles unchanged - the runtime handles replication
  • Entity id determines which region "owns" it (consistent hashing)
  • Views project locally per region
  • Workflows resume in the region where they started
