Runbook example

Alert: OrderWorkflow stuck > 5min.

Steps
1. Verify:
   akka logs --grep "workflow" --tail 100
   Open console: which workflow instance? Which step?

2. Diagnose:
   akka services describe akka-workshop
   Look for InventoryEntity or PaymentEntity errors.

3. Common causes:
   - Payment provider outage (check status.stripe.com)
   - Inventory reserved-orders leak (see runbook 61)
   - Model provider timeout (check /metrics)

4. Mitigate:
   - Transient: wait for provider recovery
   - Persistent: transition workflow to compensate:
     akka services command akka-workshop \
       order-workflow <id> forceCompensate

5. Escalate if:
   > 20 workflows stuck OR provider outage > 15min
  • Specific alert name, no ambiguity
  • Verify step confirms it's real
  • Bounded diagnostic checks
  • Concrete mitigation commands
  • Clear escalation trigger
  • Living doc — update after every incident
