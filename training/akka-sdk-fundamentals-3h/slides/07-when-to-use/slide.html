Event sourcing

Great tool. Not the only tool.

Reach for it when
  • History matters — audit, compliance, disputes
  • Multiple read models over the same domain
  • "What did X look like on Tuesday?" is a real question
  • Domain has a natural event vocabulary
Skip it when
  • State is a settings page — latest is all you need
  • You have no domain narrative, just CRUD
  • Aggregate would grow an unbounded event stream you'll never trim
  • Team hasn't internalized eventual consistency yet
Rule of thumb: if who changed what, when, and why matters — event source it.
