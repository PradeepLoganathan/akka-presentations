Endpoints

How this differs from a Spring controller.

Spring MVC / Boot
  • @RestController → @Service → @Repository → DB
  • Transactions span layers; state is in the DB
  • Concurrency is your problem
  • Distribution is your problem
Akka SDK
  • Endpoint → ComponentClient → entity or view
  • State lives in the entity; the runtime persists it
  • Single-writer per entity ID — no locking
  • Sharding, replication, retries: runtime concerns
Same three-layer feel. Different guarantees underneath.
