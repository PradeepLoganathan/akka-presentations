Commands & events

Commands are intent.

A command is a request to change state. The entity decides whether to accept it.

domain/commands/CustomerCommands.java
public final class CustomerCommands {

  public record CreateCustomer(
      String firstName,
      String lastName
  ) {}

  public record DeleteCustomer() {}
}
  • Named in the imperative - CreateCustomer, not CustomerCreated
  • Present tense: "do this"
  • Plain Java records: the field list is the contract
  • Can be rejected: "Customer already exists"
  • Never stored. They produce events, then vanish.
