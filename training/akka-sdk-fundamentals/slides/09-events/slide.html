Commands & events

Events are facts.

An event describes something that already happened. Never rejected, never edited.

domain/events/CustomerEvents.java
public sealed interface CustomerEvents {

  record customerCreated(
      String customerId,
      String firstName,
      String lastName
  ) implements CustomerEvents {}

  record customerDeleted(
      String customerId,
      String firstName,
      String lastName
  ) implements CustomerEvents {}
}
  • Past tense - customerCreated
  • Immutable. Journaled forever.
  • sealed interface → the reducer's switch is exhaustive
  • Serialized to storage - treat schema changes with care
  • Downstream views & consumers subscribe to them
