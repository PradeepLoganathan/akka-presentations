Event-sourced entity: the log is the truth.

Command handlers persist events; state is rebuilt by folding them.

domain/entities/CustomerEntity.java
@ComponentId("Customer")
public class CustomerEntity
    extends EventSourcedEntity<CustomerState, CustomerEvents> {

  @Override
  public CustomerState emptyState() {
    return CustomerState.empty();
  }

  public Effect<String> create(CreateCustomer cmd) {
    if (currentState().customerId() != null)
      return effects().error("Customer already exists");

    var id = commandContext().entityId();
    var ev = new customerCreated(id, cmd.firstName(), cmd.lastName());
    return effects().persist(ev).thenReply(__ -> id);
  }
}
  • Extends EventSourcedEntity<State, Event>
  • @ComponentId = stable identifier used for storage
  • Validate first, then .persist(event)
  • commandContext().entityId() = which instance this is
  • State is not mutated here - the reducer does that
