applyEvent — the reducer.

One method. One event in, one new state out. Called on every new event and on every replay.

CustomerEntity.applyEvent
@Override
public CustomerState applyEvent(CustomerEvents event) {
  return switch (event) {
    case customerCreated e ->
      new CustomerState(
          e.customerId(), e.firstName(), e.lastName(), false);
    case customerDeleted e ->
      new CustomerState(
          e.customerId(), e.firstName(), e.lastName(), true);
  };
}
  • Pure function. No I/O, no side effects.
  • Runs during command handling and during restart replay
  • Sealed events + switch expression = compiler-checked exhaustive
  • Return a new record — states are immutable
  • Adding an event? Adding a switch case is required by the compiler.
If applyEvent is not pure, replay after a restart will diverge from live behavior.
