Views

Views are the read side.

Entities are optimized for writes and single-ID lookups. Views answer "list all customers" without hammering every entity instance.

Subscribes

To an entity's events (event-sourced) or state (value).

Projects

Each event updates rows in a queryable table.

Serves queries

@Query methods return rows filtered by criteria.

Multiple views can subscribe to the same events. Different shape, same source of truth — classic CQRS.
