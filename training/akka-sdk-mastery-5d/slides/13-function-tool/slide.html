@FunctionTool — a method the model can call.

Anatomy of a tool method
@FunctionTool(description =
    "Look up a customer's profile by customerId. " +
    "Returns first name, last name, deleted flag.")
public String getCustomer(
    @Description("The customer id, e.g. cust-001")
    String customerId) {
  var s = client.forEventSourcedEntity(customerId)
      .method(CustomerEntity::get).invoke();
  if (s.customerId() == null)
    return "No customer found for " + customerId;
  return String.format(
      "customerId=%s firstName=%s deleted=%s", ...);
}
  • description is what the LLM reads to pick a tool
  • @Description on params helps the LLM fill them right
  • Return String — that's what the model sees
  • Format the return so the model can quote it later
  • "No data" is a valid answer; never fabricate
