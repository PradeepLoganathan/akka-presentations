Agents

Multi-agent systems, without a framework.

Multiple agents in the same service, same runtime. Sharing sessions if they should, isolated if they shouldn't.

Same session

Agents share memory. A "triage" agent hands off to a "billing" agent, both see the transcript.

Different sessions

Isolated. Useful for tenant separation or batch workers.

AgentRegistry

@AgentDescription makes agents discoverable so a supervisor agent can list & delegate.

Multi-agent lives in the same Java service — no bus, no coordinator, no separate deployment.
