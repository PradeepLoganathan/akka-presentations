Testing

Contract tests: events are forever.

An event journaled today gets replayed by code you'll write in 2028. Break the schema, break the past.

Schema tests

Fixture files of historical event JSON. Assert the new record types can deserialize each one.

Replay tests

Given a corpus of events, assert the current state matches the expected snapshot.

Migration tests

When you must break, migrate: old event → new event via a translator. Test both sides.

Rule: adding a field is safe (with a default). Renaming, removing, changing type — assume it will break replay, and test accordingly.
