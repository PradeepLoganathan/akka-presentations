Testing

Idempotency: the non-negotiable.

At-least-once delivery means "duplicates happen." Every command handler, every consumer, every timer target needs a test that says: "the second time is a no-op."

Entity commands

Send the same command twice. Assert the second attempt is rejected or idempotent.

Consumer handlers

Feed the same event twice. Assert the downstream state is identical to feeding it once.

Timer targets

Test the command the timer calls, not the timer. Idempotence is the target's problem.

A test that only fires the command once tells you nothing about production. Fire it three times.
