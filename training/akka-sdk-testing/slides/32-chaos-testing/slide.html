Testing

Chaos: assert what survives.

Not "does it work?" but "what happens when it doesn't?" Test the runtime's failure semantics on your workflow.

Mid-step kill

POST an order, watch it get to charge, kill the JVM, restart, assert SHIPPED.

Consumer redelivery

Kill mid-consume; assert the same event is redelivered on restart; assert idempotency.

Provider outage

Point OpenAI base URL at a broken host; assert onFailure handler engages, workflow doesn't corrupt.

Chaos tests are expensive to run and priceless when a production incident matches the scenario.
