Testing

Staging is not a test environment.

It's a rehearsal. What runs in staging should be identical to production - same runtime, same config, real data shapes.

CI tests

Everything in the pyramid up through integration. No network egress. Deterministic.

Staging

Full deploy. Real LLM keys (with lower quotas). Smoke tests. Chaos exercises on schedule.

Production canary

Fraction of traffic to the new version. Real monitoring. Real rollback path.

If staging finds bugs that CI didn't, add the missing CI test. Staging is the last check, not the first.
