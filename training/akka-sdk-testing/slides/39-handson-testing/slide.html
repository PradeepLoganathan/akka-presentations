Hands-on · 60 minutes

Grow the sample's test suite.

The sample repo has almost no tests. Pick one component and give it a real one.

Terminal
# still on the agents branch
mkdir -p src/test/java/com/pradeepl/akkakata

# start with the highest-risk component
touch src/test/java/com/pradeepl/akkakata/OrderWorkflowTest.java

# run
mvn test
  • Choose: OrderWorkflow, CustomerAuditConsumer, or SupportAgent
  • Write one happy-path and one failure test
  • Add an idempotency test - feed the input twice
  • Add a contract test - deserialize an old event fixture
  • Bonus: add a simple k6 script for OrderEndpoint load
