Hands-on · 30 minutes

Check out the consumers branch.

Adds CustomerAuditConsumer + AuditLogEntity on top of workflows.

Terminal
git checkout consumers
mvn compile exec:java

# create a customer
curl -XPOST http://localhost:9000/api/customers \
  -H 'Content-Type: application/json' \
  -d '{"customerId":"c-audit","FirstName":"Grace","LastName":"Hopper"}'

# delete them
curl -XDELETE http://localhost:9000/api/customers/c-audit

# see both actions in the audit log
curl http://localhost:9000/api/audit/customers/c-audit
  • Notice: no direct call to the audit from the endpoint
  • Consumer sees the event and appends asynchronously
  • Try: add an OrderAuditConsumer subscribing to OrderEvents
  • Try: publish audit events to a topic via produce()
  • Try: kill the JVM between create and delete — consumer still catches up
