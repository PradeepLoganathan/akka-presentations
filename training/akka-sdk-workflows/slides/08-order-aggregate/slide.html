Introducing the Order aggregate.

Second event-sourced entity in the sample. Six events cover the full lifecycle: placed, reserved, paid, shipped, failed, cancelled.

domain/events/OrderEvents.java
public sealed interface OrderEvents {
  record orderPlaced(...)         implements OrderEvents {}
  record inventoryReserved(...)   implements OrderEvents {}
  record paymentRecorded(...)     implements OrderEvents {}
  record orderShipped(...)         implements OrderEvents {}
  record orderFailed(...)          implements OrderEvents {}
  record orderCancelled(...)       implements OrderEvents {}
}
  • Each event = a step in the lifecycle
  • Terminal states: SHIPPED, FAILED, CANCELLED
  • Sealed = compiler-checked reducer
  • OrderEntity has command handlers to persist each event
  • The workflow decides which events fire, in what order
