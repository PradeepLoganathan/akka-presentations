Two states, distinguished

Workflow state is not entity state.

The workflow tracks orchestration context. The Order entity tracks domain truth. Same orderId, different concerns.

OrderWorkflow.State
  • What is this workflow instance working on?
  • orderId + customerId + lines + total
  • Persisted per workflow instance
  • Lost when workflow ends or is deleted
OrderState (entity)
  • What is the truth of this order?
  • orderId + lines + total + status
  • Persisted as an event log; never deleted
  • Answers "what happened to this order?"
Workflow state = what the orchestrator needs to keep going. Entity state = what history demands to be true.
