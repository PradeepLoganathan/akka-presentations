Consumers

At-least-once means "sometimes twice."

Consumers can see the same event twice if a failure happens between processing and acknowledgement. Design for idempotency.

Idempotent keys

Use the event's unique offset or subject as part of the row/entity id.

Check before write

"If already appended, skip." Guard the target entity's command handler.

Idempotent side effects

PUTs > POSTs. Upserts > inserts. Named calls > anonymous.

If you can't make it idempotent, at least make the duplicate harmless. Log twice; don't email twice.
